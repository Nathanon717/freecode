# Eval menu: menu artifacts visible above first eval output

**File:** `src/cli/eval-menu.ts`

When the eval menu had fewer items than the available screen rows, `extraCleanup` in `runRawPicker` moved the cursor up only `rowCount` rows from the controls row (row `r-2`), not reaching row 1. The `\x1b[J` erase then cleared only from the middle of the screen down, leaving menu item rows at the top intact while eval output started printing partway down.

Fixed by adding `process.stdout.write('\x1b[1;1H\x1b[J')` + `drawFooter()` in `runEvalMenuBody` immediately after the cancel guard and before dispatching to the run loop. This fires exactly once ‚Äî at the menu‚Üírun boundary ‚Äî so sequential evals in run-all mode continue appending without any between-eval clears.
